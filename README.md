# hz-openapi-gen

Generate [CloudWeGo Hertz](https://www.cloudwego.io/docs/hertz/) handlers,
routers, and models directly from an OpenAPI 3.x document.

OpenAPI is the source of truth. The spec is **not** routed through `hz`,
`thriftgo`, or `protoc` — it is parsed with
[`pb33f/libopenapi`](https://github.com/pb33f/libopenapi) and turned straight
into idiomatic, boring Go you can read and maintain.

This is a prototype: it proves the OpenAPI-first → Hertz path end to end for
common REST APIs. See [Limitations](#limitations).

## Install / run

```sh
go run ./cmd/hz-openapi-gen -spec=api/openapi.yaml -out=. -module=example.com/service
```

Or add a directive to your service and run `go generate ./...`:

```go
//go:generate go run gitlab.wildberries.ru/travel/travel-backend/adv/core/hz-openapi-gen/cmd/hz-openapi-gen -spec=api/openapi.yaml -out=. -module=example.com/service
```

## What it generates

```
example-service/
  biz/
    router/register.go     # GeneratedRegister(h *server.Hertz) with every route
    handler/{tag}.go       # handler stubs grouped by OpenAPI tag (501 not implemented)
    model/{schema}.go      # one struct per component schema
  main.go                  # optional (-generate-main); created if missing
  generate.go              # optional; carries the go:generate directive
```

Router and model files start with `// Code generated by hz-openapi-gen. DO NOT EDIT.` —
regeneration replaces them entirely when that marker is present (unless `-force` skips the check).

Handler files start with `// Code generated by hz-openapi-gen.` (without `DO NOT EDIT`) and
are generated as **extension points**: the generator creates them once, then on later runs
preserves any custom code and appends only missing handler functions. A handler file without
the generated marker is treated as hand-written and refuses to merge unless `-force` is set.

Optional `main.go` and `generate.go` are create-only files: they are written once and skipped
when already present, regardless of markers.

## Flags

| Flag | Default | Description |
| --- | --- | --- |
| `-spec` | _(required)_ | OpenAPI 3.x YAML/JSON file |
| `-out` | `.` | output root directory |
| `-force` | `false` | skip marker checks; handler files are still merged, not overwritten |
| `-package-base` | `biz` | base package folder |
| `-dry-run` | `false` | print the plan, write nothing |
| `-validate-only` | `false` | validate the spec, write nothing |
| `-generate-main` | `false` | write a minimal `main.go` if missing |
| `-generate-go-generate` | `true` | write `generate.go` with the regen directive |
| `-strict` | `true` | missing `operationId` / unsupported constructs are errors |
| `--version` | _(flag)_ | print version and exit |

Exit codes: `0` ok · `1` parse/validation/generation error · `2` usage error ·
`3` unsafe overwrite prevented.

## How it works

`load → extract → map → render → write`

- **load** (`internal/openapi`): `libopenapi.NewDocument` + `BuildV3Model`.
- **extract**: walk paths and `components.schemas` in source order into a
  resolved `SpecModel` (Go types, names, and Hertz paths all computed here).
- **map**: OpenAPI schema → Go type (`{id}` path params → `:id`; `operationId`
  → exported handler name with initialisms preserved, e.g. `getUserById` →
  `GetUserByID`).
- **render** (`internal/generator`): `text/template`, then `hzutil.FormatGo` with a
  hard failure on invalid Go.
- **write** (`internal/writer`): an all-or-nothing plan with three write modes —
  `cover` (router/models), `create` (main/generate), and `merge` (handlers).

## Development

```sh
make test-unit       # unit + golden tests, skip compile gate
make build           # compile source packages
make lint            # validate config and run lint/static-analysis/security
make vulncheck       # reachable vulnerability scan
make test-generated  # full generated-service compile gate
make golden          # refresh golden snapshots
```

GitLab CI runs the same gates as parallel quality jobs: build, short tests, generated-service compile, lint, and vulnerability scan.

## Limitations

Prototype scope (deferred, not broken):

- OpenAPI **3.x only** (Swagger 2.0 is rejected).
- JSON request/response bodies only; non-JSON media types are ignored.
- `oneOf`/`anyOf` → `interface{}` (with a warning); `allOf` is not flattened yet.
- Inline (non-`$ref`) request bodies and nested inline objects are not modeled
  as named structs.
- Path/query parameters are extracted (with Hertz bind tags) but not yet
  assembled into request structs; handlers bind the JSON body only.
- No business logic, auth, middleware, or database code is generated. Handlers
  return `501 not implemented` — never a fake `200`.
